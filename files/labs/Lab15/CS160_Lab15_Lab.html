CSCI 160 Lab 15

The Towers of Hanoi is a game/puzzle with origins lost in the mists of time. Its legend is that in a Buddhist monastery in Hanoi, Vietnam, the monks work tirelessly moving disks from one tower to another, following the rules of the game; when they finish moving all the disks to the last tower, the world will end.

The puzzle layout consists of three spindles, the first of which has n disks stacked on it, each disk having a smaller diameter than the one below it, forming a stepped cone. The rules are simple:

  • Only one disk may be moved at a time.
  • Each move consists of taking the upper disk from one of the rods and sliding it onto another rod, on top of the other disks that may already be present on that rod.
  • No disk may be placed on top of a smaller disk.

The puzzle has an elegant recursive solution, which you will be implementing in today's lab.

Part 1: Building the tower components.

The files you are given include the TowerDisk class and the Tower class. The Tower class is composed of a base (GRect), a spindle (GRect), and TowerDisk objects. You should start by implementing the constructor for the TowerDisk class. This constructor is recursive in the sense that it will draw the rectangle which represents itself, then create the next TowerDisk object, much like how each WormSegment drew itself, then created the next WormSegment object in the previous lab.

Once you have the constructor, implementing the remaining methods should be fairly straightforward. Work on this class until you no longer have any compiler errors.

Once you can compile without errors, use the TowerTest program to see what your tower looks like. The stack of disks should look like a pyramid with a small amount of space between disks; the base disk should be below the colored disks; and the spindle should appear to pass through the center of the disks and stick out the top a little. To see a sample solution, change into the solution subdirectory and run java TowerTest.

When you are satisfied with your tower, show it to the TA or lab instructor before moving on.

 

Part 2: Solving the puzzle

The remaining task is to implement the recursive Towers of Hanoi algorithm. The basic algorithm is as follows:

  • If there's only one disk to move, move it directly from the source to the destination.
  • Otherwise, move the stack of n − 1 disks recursively from the source to the intermediate tower, using the destination as an intermediate; move the remaining disk to the destination; and finally move the stack of n − 1 disks recursively from the intermediate tower to the destination, using the source as an intermediate.

Again, see the above "Understanding the Solution" section for written descriptions and videos of the solution.

The code for the algorithm goes in the first move() method in the TowerOfHanoi class.  This is the only method that you should need to modify.  However, it relies heavily on your implementation of TowerDisk from Part 1 (specifically, methods addDisk() and removeDisk()), so you may need to make some corrections there if the animations don't quite work.

Test your algorithm using the TowersOfHanoi program. Again, you can see a sample in the solution subdirectory. In the sample and your completed solution, you should see the disks move between the towers, without violating any of the rules, until the final result where the disks are stacked on the last tower.

Once you have a working solution, show your project to the TA or lab instructor.  You've done it!  (Does that mean that the world is about to end?)
