Part 1: Keeping track of who's busy and who's idle in the steno pool

In the old days, there were no personal computers, no iPods, no Dictaphones (and no Internet, but that's not relevant here). If a business person wanted to create a document, such as sending a letter, they wouldn't create it themselves—oh no!—they would use a stenographer. A stenographer took dictation using shorthand and then transcribed the shorthand into a document using a typewriter. In a large company, business people who weren't important enough to have their own personal secretary would request someone from the steno pool, a group of stenographers who were available to take dictation and type documents for the business people. Ask your grandparents if you want to know more, or just watch Mad Men.

Your job is to write a program that manages a steno pool. The pool should be implemented using two sets, one for the available stenographers and one for those stenographers currently taking dictation. Just use names (Strings) for the stenographers.

The program will handle several operations:

  • add stenographers to the available set
  • remove them from the available set
  • list the stenographers in each set separately
  • move them between the two sets
  • quit the program.

Show your work to the TA or lab instructor before you go on.

Part 2: Keeping track of inventory

In this part, you will write a program that manages an inventory list. You should use the command interpreter from the previous exercise as a model. Each inventory item, represented by the opaque class Part, will have an int part number, a String name and an int number of items of this part in stock.

The class Part has the following methods defined:

  • Part(int partNumber, String partName);
  • int getPartNumber();
  • int getPartInStock();
  • String getPartName();
  • void setPartInStock(int);
  • String toString();

This program will use one map for the part number, part name, and stock. It will be of type Map<Integer, Part>

The program will handle several operations:

  • Add new inventory items with an initial stock of 0. Report to the user if the item couldn't be added because the part number is already in use.
  • Look up an inventory item by part number and report the part name and stock.
  • Remove an inventory item by part number.
  • Add stock for an inventory item. Be sure not to add stock to an item that isn't already in the map.
  • Remove stock for an inventory item, but not more than the stock on hand.
  • Display the entire inventory.
  • Save the inventory to a file, one record per line, in the format: part number<newline>part name<newline>stock
  • Load a new inventory from a file in the same format. (Done for you)
  • Quit the program.

When you have finished, show your work to the TA or lab instructor.
